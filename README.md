# 期货研究工具

一个可本地启动、可通过浏览器访问的中文期货研究门户原型。

## 启动方式

```bash
python3 app.py
```

启动后访问 `http://127.0.0.1:8000`。

如需安装项目依赖，直接执行：

```bash
pip install -r requirements.txt
```

当前环境里已检测到的版本只有 `pandas==2.3.3`。
`akshare`、`arch`、`statsmodels`、`yfinance` 在这台机器上还未安装，所以 `requirements.txt` 中对它们使用的是兼容下限；等你实际安装并验证后，如果你想，我可以再帮你把它们改成严格锁定版本。

## 当前功能

- 单页布局：上方建模区、下方研究区
- 建模区的`期货产品`会同时控制整页的研究对象
- 建模区可选择模型并发起本地预测任务
- 预测任务状态会显示“模型运算中”与运行计时
- 任务完成后状态会提示前往研究区检查结果
- 预测结果会写入本地 SQLite：`data/predictions.db`
- 日线行情会写入同一个 SQLite 中的 `market_data` 表
- 研究区提供日线 K 线占位图，可叠加多个模型的最近一次预测结果
- 右侧参数区是可滚动多选列表，支持 MA、布林带和模型结果勾选
- 每个模型在参数区会显示该模型最近一次更新时间

## 说明

本轮已经补上独立的预测运行器、`models/` 目录中的 ARIMA/GARCH 模型，以及 `data_extraction.py` 数据提取模块。
数据提取优先读取本地 SQLite 的日线数据，并在需要时尝试通过 AkShare 与 Yahoo Finance 刷新。

## 模型预测方法说明

### 1) ARIMA 基线模型

- 输入：历史收盘价序列。
- 主路径：先将价格转为对数收益率，再使用 ARIMA 对收益率做预测；将预测收益率还原为未来价格路径。
- 增强项：叠加残差振荡项，避免过于平滑或机械的直线预测。
- 回退逻辑：当样本不足或建模失败时，切换到“趋势 + 波动 + 周期项”的启发式预测。
- 风险约束：单步收益有上下界限制，避免极端跳变。

### 2) GARCH 波动率模型

- 输入：历史收盘价序列。
- 主路径：
  - 使用 AutoReg 估计收益率均值部分；
  - 使用 GARCH 估计条件方差（波动率）；
  - 将均值项与波动冲击组合，重建未来价格路径。
- 回退逻辑：若 `arch` 或拟合过程不可用，切换到“均值收益 + 波动扰动 + 周期项”的替代路径。
- 目标：显式利用波动率信息，使预测路径更贴近“有波动聚集”的市场特征。

### 3) 多模型综合系统（Multi-Model Integrated）

- 设计思想：多路径预测 + 市场状态识别 + 动态加权 + 风险平滑。
- 输入特征（由日线构造）：收益率、对数收益率、10/20日波动率、MA10/20/60、5/20日动量、20日回撤、成交量趋势、价格偏离 MA60 等。

- 三条基础预测路径：
  - 技术路径：调用 ARIMA 路径作为短期技术预测。
  - 宏观代理路径：优先使用 SARIMAX（外生变量含动量/波动/量能趋势）预测收益，再还原价格。
  - 基本面代理路径：优先使用 OLS 拟合“未来10日收益”与特征关系，再外推到预测期。

- 市场状态识别：
  - 根据最近波动率、回撤、动量、价格偏离等指标，判定 `bull / bear / crisis / normal`。

- 动态加权融合：
  - 基础权重为技术、增强技术、基本面、宏观四部分；
  - 再按市场状态调整并归一化；
  - 其中“增强技术”是对技术路径做 regime/risk 修正后的路径。

- 风险与稳定化处理：
  - 对收益进行单步边界约束（限制极端单日变化）；
  - 对最终价格路径做区间裁剪与递推平滑，抑制离群值。

- 回退逻辑：
  - 若样本过短（如不足 40 条），直接退化为 ARIMA；
  - 若 SARIMAX/OLS 等依赖或拟合失败，自动使用启发式替代路径。

## 模型差异对比表

| 模型 | 核心思想 | 主要依赖 | 复杂度 | 速度（相对） | 适用行情 | 优点 | 局限 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| ARIMA 基线模型 | 单变量时间序列收益预测 | `statsmodels` | 低 | 快 | 趋势相对平稳、噪声中等 | 结构简单、解释性较好、调试成本低 | 对突发波动与结构切换较敏感 |
| GARCH 波动率模型 | 均值过程 + 条件方差（波动）联合建模 | `statsmodels` + `arch` | 中 | 中 | 波动聚集明显、震荡较多 | 能显式刻画波动率变化 | 参数较敏感，对样本质量要求更高 |
| 多模型综合系统 | 技术/宏观代理/基本面代理 + 状态识别 + 动态加权 | `numpy` `pandas` `statsmodels`（可回退） | 高 | 较慢 | 状态切换频繁、单一模型不稳定 | 鲁棒性更强，能融合多视角信息 | 实现复杂，调参与排错成本更高 |

> 说明：速度为同等数据规模下的相对比较（快/中/较慢），会受本机 CPU、依赖库版本与样本长度影响。
